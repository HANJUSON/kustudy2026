# 학습내용
    1. 스레드 개념: 스레드는 프로세스 내 CPU 실행 단위로, 스레드 ID·PC·레지스터·스택은 독립적으로 소유하고, 코드·데이터·파일은 같은 프로세스 내 스레드끼리 공유한다.

    2. 멀티스레딩의 이점: 반응성, 자원 공유, 경제성, 확장성의 4가지 이점을 다룬다.

    3. 멀티코어 프로그래밍과 암달의 법칙: 동시성과 병렬성을 구분하고, Speedup ≤ 1/(S +(1-S)/N) 공식을 통해 직렬 구간(S)이 성능 향상의 병목임을 설명한다.

    4. 데이터 병렬성 vs 태스크 병렬성: 데이터 병렬성은 동일 연산을 데이터를 쪼개 분산하고, 태스크 병렬성은 서로 다른 연산을 각 스레드에 분배한다. 실제로는 두 방식의 혼합 형태가 흔하다.

    5. 멀티스레딩 모델: Many-to-One, One-to-One, Many-to-Many의 세 가지 모델을 비교한다.

    6. 스레드 라이브러리: Pthreads, Windows Threads, JavaThreads 세 가지 주요 라이브러리의 API와 사용 패턴을 다룬다.

    7. Java Executor Framework: 스레드 풀로 생성·소멸 비용을 줄이고, Callable/Future로 스레드에서 결과값을 반환하는 패턴을 학습한다.

    8. Lab(Pthreads 실습): 배열 합산을 4개 스레드로 분할하는 데이터 병렬 프로그램을 작성하며, 루프 변수 직접 전달로 인한 경쟁 조건(race condition)과 공유 변수 동시 접근 문제를 직접 확인한다.
# 상세활동 내용
    수업시간에 배운 스레드 개념, 데이터 병렬성, Pthreads API, 경쟁 조건 회피 설계를 바탕으로 1000만 개의 정수 데이터를 여러 스레드로 나누어 히스토그램을 병렬로 계산하는 프로그램을 작성하였다.

    데이터 배열을 스레드 수만큼 균등하게 분할하고, 각 스레드가 자신의 구간에서 독립적으로 빈도를 카운팅하도록 hist_worker 함수를 작성하였다. 경쟁 조건을 방지하기 위해 각 스레드가 local_hist[NUM_BINS]라는 독립적인 로컬 히스토그램을 소유하도록 struct hist_arg 구조체를 설계하였고, 병합은 모든 스레드 종료 후 메인 스레드에서만 수행하였다.

    pthread_create()와 pthread_join() 루프를 분리해 진정한 병렬 실행을 보장하였으며, 루프 변수 i의 주소를 직접 전달하는 대신 args[i].id = i처럼 독립된 구조체에 값을 복사하여 안전하게 인자를 전달하였다.

    암달의 법칙을 실증하기 위해 1, 2, 4, 8개의 스레드로 스피드업을 측정하였다. 8스레드에서도 2.35배에 그쳤는데, 히스토그램 연산이 메모리 접근 위주라 메모리 대역폭이 병목으로 작용하기 때문이다. 마지막으로 verify() 함수를 통해 순차 실행과 병렬 실행 결과를 비교하는 검증 로직을 추가하였다.

    

    코드 작성과 실습을 마치고 수업시간에 배운 내용과 지금까지의 퀴즈 출제 경향을 바탕으로 다음 퀴즈 문제를 예측, 분석하고 공부하는 시간을 가졌다. 보고서 작성일에는 퀴즈를 마친 상태이고 전원 만점을 받는 쾌거를 이루었다.
