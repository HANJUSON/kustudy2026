준비 · 진행 · 개선
  • 고정 루틴 : 매주 오프라인으로 모여(3월말~6월, 회당 약 2~5시간) 개념 정리 → 코드/웹앱 구현 → 실행·검증 → 퀴즈/시험 대비 순으로 진행
  • 이론과 구현의 연결 : 매 주차 배운 개념을 반드시 "동작하는 산출물"로 구현(Closest Pair 웹앱, 병렬 히스토그램, 표절검사기, 스케줄러 시뮬레이터 …)
  • 개선 : 시험 대비 주차에는 오픈북 특성에 맞춰 교과서 정독·인덱싱 + NotebookLM 예상질문으로 전략을 전환하는 등, 매주 방식을 목표에 맞게 조정
역할 · 기준 · 과제 · 규칙
  • 역할 분배 : Frontend / Backend / LLM·Prompt / PM(일정·문서) 4개 축으로 분담(9주차 정립)
  • 결과물 기준 : 모든 주차 활동을 weekNN.md 보고서 + 실행 가능한 코드/웹앱으로 남기는 것을 원칙으로 함
  • 버전 관리 규칙 : 결과물을 팀 GitHub 레포지토리로 관리하고 주차별로 커밋하여 기록 누적
  • 퀴즈·시험 대비 규칙 : 매주 퀴즈 오답을 공유하고 다음 주 출제 경향을 함께 예측·대비(2주차 퀴즈 전원 만점)

05 · 활동 소감
팀원별 활동 소감
손한주 · 대표 / LLM 하네스
대표로서 매주 일정과 역할을 조율하며, 특히 12주차 Liberal_OS에서 AI 코딩 에이전트가 xv6 커널 개발을 자율 수행하도록 만드는 하네스(자동 테스트·회귀 게이트·작업 루프)를 설계한 경험이 가장 값졌다. AI를 개발 사이클을 굴리는 주체로 다루며 '하네스 엔지니어링'의 중요성을 체감했고, 끝까지 다듬지 못한 한계를 정직하게 기록하는 일도 실력의 일부임을 배웠다.
이태진 · 운영체제·시스템
추상적으로만 알던 운영체제 개념이 실제 코드에서 어떻게 구현되는지를 xv6 소스(proc.c, scheduler, syscall)를 직접 수정하며 확인한 점이 인상 깊었다. 시스템 콜 추가, 우선순위 스케줄러·메모리 쿼터 작업에서 저수준 디버깅의 어려움을 체감했고, 이론과 구현의 간극을 메우려면 더 깊은 시스템 이해가 필요함을 절실히 느꼈다.
송명준 · AI 활용·품질
Claude Code·Upstage Solar를 코드 리뷰·복잡도 최적화·문서화에 활용하며, 같은 도구라도 어떻게 묻고 어떻게 검증하느냐에 따라 결과 품질이 크게 달라짐을 배웠다. AI 제안을 그대로 신뢰하지 않고 테스트·벤치마크로 검증하는 습관을 들인 점이 잘한 부분이며, 효율과 정확성을 동시에 잡는 AI 활용법을 더 다듬고 싶다.
박상언 · 추천·AI 백엔드
11주차 Liberal_algorithm에서 Union-Find 군집화·Kruskal 최대신장포레스트로 취향 그래프를 만들고 그 위에 LLM 추천을 얹는 백엔드 파이프라인을 설계하며, 결정론적 수치 계산과 LLM 해석의 역할을 분리하는 설계 원칙을 체득했다. 추천 시스템·AI 백엔드의 전체 흐름을 직접 구현한 점이 값졌고, 추천 결과의 정량 평가는 더 보완이 필요하다고 느꼈다.
서로 다른 관심사(하네스·시스템·AI 활용·백엔드)를 가진 네 명이 한 주제를 각자의 시각에서 파고든 덕분에, 혼자서는 닿기 어려운 범위(웹앱부터 커널까지)를 한 학기에 함께 경험할 수 있었다.

마무리 후 활동 · 앞으로의 계획
스터디 마무리 후 활동 계획
  • 기말고사 대비를 위해 6~12주차에서 다룬 탐색 트리·동기화·저장장치·스케줄링 개념을 종합 복습하고, 각 주차 결과물을 시험 관점에서 재정리한다.
  • 스터디가 끝난 뒤에도 팀원들이 주기적으로 모여, 한 학기 동안 쌓은 알고리즘·운영체제·LLM 역량을 살려 다양한 공모전·해커톤에 꾸준히 출품하는 활동을 이어간다.
스터디를 통해 세운 앞으로의 계획
  • 최종 프로젝트 고도화 : Liberal_OS의 향후 과제(순차 baseline 벤치, 결함 격리 테스트, virtio-serial 드라이버, 우선순위 라운드로빈/에이징, 실 LLM 병렬 호출)를 다음 학기 연계 주제로 발전
  • 전공 심화(시스템 프로그래밍/분산 시스템/AI 시스템)로 학습 범위를 확장하고, 스터디 운영 방식을 다음 학기에도 지속
  • 각자의 개인 목표(LLM 하네스 · AI 서비스 백엔드 · 시스템 개발 · 효율적 AI 활용)에 맞춰 AI 기반 개발 역량을 지속적으로 심화

06 · 앞으로의 계획
팀원별 향후 계획
손한주
LLM 하네스·에이전트 오케스트레이션을 더 깊이 공부해 Liberal_OS의 live 모드 안정화·에이전트 평가 자동화를 이어가고, AI가 주도하는 개발 워크플로우 설계 역량을 개발자로서의 기반으로 다질 계획이다.
이태진
시스템 프로그래밍·운영체제 후속 학습을 이어가 virtio-serial 드라이버 같은 커널 확장 과제에 도전하고, 저수준 코드를 직접 다루는 시스템 개발자로 성장하기 위한 기초를 다질 계획이다.
송명준
AI 협업 워크플로우를 팀 표준으로 정리하고, 테스트·검증 자동화와 코드 품질 도구를 익혀 AI를 효율적이면서도 신뢰할 수 있게 쓰는 개발 습관을 굳힐 계획이다.
박상언
추천 시스템·AI 서비스 백엔드를 심화 학습해 Liberal_algorithm을 API·데이터베이스를 갖춘 실제 서비스 형태로 확장하고, LLM 기반 애플리케이션 개발자로 나아갈 계획이다.
다음 학기에도 같은 팀으로 스터디를 이어가며, 이번에 만든 두 프로젝트(Liberal_algorithm·Liberal_OS)를 후배·동료에게 공유하는 발표·문서화 활동을 진행할 계획이다.
