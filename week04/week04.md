# 학습내용
    CPU 스케줄링은 준비 큐(Ready Queue)에 있는 프로세스들 중 어떤 프로세스에게 CPU를 할당할지 결정하는 운영체제의 핵심 기능이다. 스케줄링 알고리즘의 성능은 평균 대기 시간(Average Waiting Time), 평균 반환 시간(Average Turnaround Time), CPU 이용률, 응답 시간 등의 지표로 평가한다.
    FCFS(First-Come, First-Served) 알고리즘은 도착 순서대로 CPU를 할당하는 비선점형 방식이다. 구현이 단순하고 직관적이지만, 실행 시간이 긴 프로세스가 먼저 도착하면 이후 짧은 프로세스들이 오래 대기해야 하는 호위 효과(Convoy Effect)가 발생한다. 예를 들어 P1(10ms), P2(4ms), P3(6ms), P4(2ms), P5(8ms)가 순서대로 도착하면 P4는 실제 실행 시간(2ms)보다 훨씬 긴 19ms를 대기하게 된다. 시뮬레이션 결과 평균 대기 시간은 9.4ms, 평균 반환 시간은 15.4ms로 세 알고리즘 중 가장 높았다.
    SJF(Shortest Job First) 알고리즘은 준비 큐에서 CPU 버스트 시간이 가장 짧은 프로세스를 먼저 실행하는 방식이다. 평균 대기 시간을 이론적으로 최소화하는 최적 알고리즘으로, 동일한 프로세스 집합에서 평균 대기 시간 4.0ms, 평균 반환 시간 10.0ms를 기록하였다. 그러나 실제 운영체제에서는 다음 CPU 버스트 시간을 사전에 알 수 없다는 현실적 제약이 있으며, 긴 프로세스가 무한정 대기하게 되는 기아(Starvation) 문제가 발생할 수 있다. 실제 구현 시에는 지수 평균(Exponential Averaging)을 통한 예측 기법으로 근사한다.
    CPU 스케줄링 알고리즘이 사용되는 대표적인 예시 3가지에는
    FCFS(First-Come, First-Served): 도착 순서대로 CPU를 할당하는 비선점형 알고리즘이다. 구현이 단순하지만 호위 효과로 인해 짧은 프로세스가 오래 대기할 수 있다. 대형 배치 처리 시스템처럼 응답 시간보다 처리량이 중요한 환경에 적합하다.
    SJF(Shortest Job First): 버스트 시간이 짧은 프로세스를 우선 실행하여 평균 대기 시간을 최소화한다. 인쇄 스풀러, 배치 작업 스케줄러처럼 작업 길이를 예측할 수 있는 환경에서 유용하다. 기아 문제를 방지하기 위해 에이징(Aging) 기법과 함께 사용된다.
    Round Robin(RR): 모든 프로세스에 동일한 타임 퀀텀(Time Quantum)을 순서대로 부여하는 선점형 알고리즘이다. 응답 시간이 균등하고 기아 문제가 발생하지 않아, 시분할 시스템(Time-Sharing System)과 대화형 운영체제에 가장 널리 사용된다. 타임 퀀텀 q=4ms 설정 시 평균 대기 시간 7.2ms, 평균 반환 시간 13.2ms를 기록하였으며, 퀀텀 크기에 따라 성능이 크게 달라진다.
# 상세활동 내용
    수업시간에 배운 CPU 스케줄링 개념을 바탕으로 프로세스의 도착 시간과 버스트 시간을 입력받아 FCFS, SJF, Round Robin 알고리즘의 동작을 시뮬레이션하는 웹앱 프로그램을 작성하였다. 각 알고리즘별 Gantt Chart와 평균 대기 시간, 평균 반환 시간을 자동으로 도출하여 성능을 정량적으로 비교하는 방식으로 설계하였다.

    입력된 프로세스 집합(P1~P5, 도착 시간 및 버스트 시간 각각 상이)을 기반으로 각 알고리즘을 순차적으로 적용하였다. FCFS는 도착 순서대로 큐에 삽입하여 비선점 방식으로 처리하였고, SJF는 매 시점 준비 큐에서 버스트 시간이 가장 짧은 프로세스를 선택하도록 구현하였다. Round Robin은 타임 퀀텀 q=4ms로 설정하고 원형 큐(Circular Queue) 구조로 프로세스를 순환 처리하였다. 각 스텝마다 현재 실행 프로세스, 대기 시간, 반환 시간을 누적하여 최종 평균값을 산출하도록 구현하였다.

    프로그램을 완성한 후 P1(도착 0ms, 버스트 10ms), P2(도착 1ms, 버스트 4ms), P3(도착 2ms, 버스트 6ms), P4(도착 3ms, 버스트 2ms), P5(도착 4ms, 버스트 8ms)를 입력하여 실행하였다. 그 결과 평균 대기 시간은 FCFS 9.4ms, SJF 4.0ms, Round Robin 7.2ms가 나왔고, 평균 반환 시간은 각각 15.4ms, 10.0ms, 13.2ms로 SJF가 가장 우수한 성능을 보였다.
