> **알고리즘 × 운영체제 × LLM** — 강의에서 배운 이론을 매주 '동작하는 코드'로 구현한 한 학기의 기록
>
> 고려대학교 과학기술대학 컴퓨터융합소프트웨어학과 · KUS-TUDY 학습공동체 **Big O(S)** 팀 (손한주 · 이태진 · 송명준 · 박상언)

## 0. 레포지토리 설명

이 레포지토리는 KUS-TUDY 학습공동체 **Big O(S)** 팀의 **2026학년도 1학기 스터디 활동 전체 기록**입니다.

우리 팀의 목표는 단순한 문제 풀이를 넘어, **알고리즘**과 **운영체제** 두 전공 강의에서 배운 핵심 개념을 매주 곧바로 **실제로 동작하는 프로그램**으로 구현하는 것이었습니다. `이론 → 구현 → 검증`의 사이클을 매주 반복했고, 학기 후반에는 LLM(Upstage Solar)을 결합한 두 개의 팀 프로젝트로 학습 성과를 집약했습니다.

레포에는 다음이 담겨 있습니다.

- **주차별 활동 보고서** (`week01` ~ `week12`) — 학습목표·학습내용·상세활동 기록
- **주차별 결과물** — 직접 만든 웹앱·C 코드·개념 시각화 HTML
- **최종 프로젝트 2종** 링크 — [Liberal_algorithm](https://github.com/HANJUSON/Liberal_algorithm) · [Liberal_OS](https://github.com/HANJUSON/Liberal_OS)
- **우수그룹 자기추천서 자료** — `draft.md`(텍스트 초안) + `자기추천서_발표자료.html`(발표자료)

> 팀명 **Big O(S)** 는 점근 표기 **Big-O**와 **O(S) = OS(운영체제)**를 겹쳐, 두 과목을 함께 공부한다는 뜻을 담았습니다.

## 2. 파일 구조

```
Kustudy/
├── README.md                       # 이 문서
├── draft.md                        # 우수그룹 자기추천서 초안 (텍스트, 발표자료의 기반)
├── 자기추천서_발표자료.html         # 자기추천서 발표자료 (브라우저로 열기 · ← → 로 이동)
├── DESIGN-korea-university.md       # 발표자료 디자인 가이드 (고려대학교 스타일)
│
├── week01/                         # ── 주차별 활동 폴더 (week01 ~ week12) ──
│   ├── week01.md                   #   주차별 활동 보고서 (학습목표 · 학습내용 · 상세활동)
│   ├── KakaoTalk_*.png / *.jpg     #   활동 사진(모임) + 결과물 캡처
│   ├── result/                     #   그 주차에 만든 웹앱 (app.py + static/)
│   │   ├── app.py
│   │   └── static/ (index.html, style.css, app.js)
│   └── image01.png ...             #   실행 화면 등
│
├── week02/  (histogram.c · 활동 사진)
├── week04/  (matmul.c · mergesort.c · 시뮬레이터 .html · 사진)
├── week06/ ~ week10/  (개념 시각화 *.html · 활동/증빙 사진)
├── week11/  (Liberal_algorithm 실행 화면 다수)
├── week12/  (Liberal_OS 실행 화면)
└── ...
```

**각 `weekNN` 폴더 공통 구성**

| 파일 | 설명 |
|---|---|
| `weekNN.md` | 그 주차 활동 보고서 — **무엇을 배우고 무엇을 만들었는지**가 모두 적혀 있음 |
| 사진(`.png`/`.jpg`) | 스터디 모임 사진(활동) + 프로그램 실행 결과(증빙) |
| `result/` · `*.c` | 그 주차에 직접 만든 웹앱 / 코드 |
| `*.html` | 개념을 시각화한 결과물 — **브라우저로 열면 바로 실행**됨 |

## 3. 우리가 한 활동 (한눈에 보기)

매주 **강의 진도 → 개념 정리 → 직접 구현(코드/웹앱) → 실행·검증 → 퀴즈·시험 대비** 순서로 진행했습니다.

| 주차 | 주제 | 한 일 (요약) | 결과물 |
|:--:|---|---|---|
| 1 | 분할 정복 | 가장 가까운 두 점 쌍(Closest Pair)을 완전탐색 vs 분할정복으로 비교하는 웹앱 | 웹앱 |
| 2 | 스레드·병렬·암달의 법칙 | 1,000만 정수 히스토그램을 멀티스레드로 병렬 계산, 스레드 수별 속도 측정 | `histogram.c` |
| 3 | 동적 계획법(DP) | LCS(최장 공통 부분 수열)로 문서 유사도를 재는 **표절 검사기** 웹앱 | 웹앱 |
| 4 | CPU 스케줄링 | FCFS / SJF / Round Robin을 간트 차트로 보여주는 **시뮬레이터** | 웹앱·HTML |
| 5 | 중간고사 대비 | 알고리즘·운영체제 핵심 개념 정리 + 예상 문제 풀이 | 시험 정리본 |
| 6 | 탐색 트리 | BST → 레드-블랙 트리 → B-트리의 균형 원리 비교 실습 | 시각화 HTML |
| 7 | 동기화 | 경쟁 상태·임계구역, 세마포어/모니터, 고전 3문제(식사하는 철학자 등) | 시각화 HTML |
| 8 | V 모델 | 요구사항 명세가 왜 중요한지 알고리즘·동기화 사례로 분석 | 시각화 HTML |
| 9 | 팀 프로젝트 킥오프 | 데이터 분석 웹앱 코드 정독 + LLM 결합 아이디어 기획 | 주제 기획 |
| 10 | 대용량 저장장치 | HDD/SSD 구조, 디스크 스케줄링, RAID 레벨 비교 | 시각화 HTML |
| 11 | 알고리즘 + LLM | 다양한 알고리즘을 직접 구현하고 LLM 추천을 얹은 프로그램 | **[Liberal_algorithm](https://github.com/HANJUSON/Liberal_algorithm)** |
| 12 | xv6 멀티에이전트 OS | xv6 커널을 직접 수정해 LLM 에이전트를 운영하는 운영체제 | **[Liberal_OS](https://github.com/HANJUSON/Liberal_OS)** |

### 대표 결과물

- **[Liberal_algorithm](https://github.com/HANJUSON/Liberal_algorithm)** — Heap Sort·이진 탐색·LRU 캐시·Union-Find·Kruskal 등 알고리즘을 직접 구현하고, 수치 계산(결정론)과 LLM(해석·추천)의 역할을 분리한 추천 프로그램
- **[Liberal_OS](https://github.com/HANJUSON/Liberal_OS)** — 5개의 LLM 에이전트를 xv6 사용자 프로세스로 띄우고, 스케줄링·동기화·시스템 콜·검증/캐시를 **직접 수정한 xv6 커널**에 구현한 "LLM을 위한 운영체제"

### 처음 보는 분이라면

1. **그 주에 뭘 했는지 궁금하면** → 해당 `weekNN/weekNN.md` 를 여세요. 모든 내용이 글로 정리돼 있습니다.
2. **결과물을 직접 보고 싶으면** → `weekNN/` 폴더의 `*.html` 파일을 **브라우저로 더블클릭**하면 바로 실행됩니다.
3. **활동 전체를 한 번에 보고 싶으면** → `자기추천서_발표자료.html` 을 브라우저로 열고 **← → 키**로 넘기세요. (한 학기 활동을 17장 슬라이드로 정리)
