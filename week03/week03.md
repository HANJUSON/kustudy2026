# 학습내용
    1. DP 알고리즘은 작은 부분 문제들을 먼저 풀어내고, 풀어낸 값을 저장한 후, 이 저장된 값을 이용해 더 큰 부분 문제를 풀는 과정을 반복해 최종적으로 원 문제를 해결하는 과정까지 빌드업하는 알고리즘입니다. DP 알고리즘의 요구사항은 2가지 있는데, 첫번째는 최적 부분 구조(전체 문제의 최적해가 부분 문제들의 최적해를 포함한다)이고 두번째는 중복 부분 문제(같은 부분 문제를 반복해서 풀게 만드는 구조)이다.

    2. D&C 알고리즘의 경우 부분 문제들이 독립적이고, 탑-다운 방식(큰 값에서 작은 값으로)을 사용한다. DP 알고리즘의 경우 부분 문제들을 공유하며, 바텀-업 방식(작은 값에서 큰 값으로)을 사용한다.

    3. DP 알고리즘이 사용되는 대표적인 예시 3가지에는

    - 피보나치 수열(Fibonacci Numbers): 순수 재귀로는 시간 복잡도가 O(2^n)이 나오지만, 바텀-업 방식으로 O(n)으로 줄일 수 있다.

    - 행렬경로 문제(Matrix Path Problem): n×n 행렬에서 좌상단→우하단으로 이동할 때 합이 최대인 경로 찾기이다. 오른쪽 또는 아래쪽으로만 이동 가능하고, c[i, j] = m[i,j] + max(위, 왼쪽)으로 테이블을 채워 O(n²)의 시간복잡도를 가진다.

    - 최장 공통 부분 수열(LCS): 두 문자열에서 공통으로 나타나는 가장 긴 부분 수열을 찾는 알고리즘이다. DP 테이블을 만들고 비교하려는 문자가 같으면 대각선 +1, 다르면 위/왼쪽 값 중 최댓값을 가져온다. 예를 들어 "ABCBDAB"와 "BDCABA"에 적용하면 LCS는 "BCBA"가 나온다. 시간복잡도는 크기가 각각 m, n인 문자열을 하나하나 비교하므로 O(mn)이다.
# 상세활동 내용
    수업시간에 배운 동적 프로그래밍 개념을 바탕으로 문자열 유사도를 측정하는 최장 공통 부분 수열(LCS) 기반 표절 검사기 웹앱 프로그램을 작성하였다. 두 문서 간의 공통 부분 수열의 길이를 계산하여 유사도를 정량적으로 평가하는 방식으로 설계하였다.

    입력된 두 텍스트를 전처리(공백/특수문자 제거, 소문자 변환)한 뒤, 길이가 각각 n, m인 문자열에 대해 2차원 DP 테이블 dp[n+1][m+1]을 구성하였다. 각 셀은 부분 문자열 간 LCS 길이를 의미하며, 점화식(dp[i][j] = dp[i-1][j-1] + 1 if match, else max(dp[i-1][j], dp[i][j-1]))을 기반으로 값을 채웠다. 최종적으로 dp[n][m] 값을 이용해 유사도 비율을 계산하고, 임계값을 기준으로 표절 여부를 판별하도록 구현하였다.

    프로그램을 완성한 후에 "ALGORITHM"과 "ALTRUISTIC"이라는 문자열을 비교하는 실행을 하였다. 그 결과 LCS는 "ALRIT"가 나왔고, 유사도는 5(LCS의 크기) / max(9,10)(각각 1번째, 2번째 문자열의 크기) 를 실행해 50.0%라는 유사도가 나왔음을 도출해냈다.

    실험적으로 서로 다른 길이와 유사도를 가진 문서 쌍에 대해 성능을 측정한 결과, 알고리즘의 시간 복잡도 O(nm)의 특성상 입력 크기가 커질수록 실행 시간이 증가함을 확인할 수 있었다. 이를 통해 LCS 기반 접근법의 정확성과 한계를 함께 이해할 수 있었다.
