# 학습내용
   - 팀 구성 및 역할 분배: Frontend / Backend / LLM·Prompt / PM 4개 역할을 3~4인 팀에 어떻게 배분할지, 그리고 한 명이 복수 역할을 맡을 때의 워크로드 분산 전략을 학습.
   - 점수 산정 파이프라인 분석(app.py): 활동 가중치 합산 → Recency Decay(지수 감쇠 e^(-λd)) → Min-Max 정규화 → Binary Search 그룹 분류 → Heap Sort 정렬 → LRU 캐시 까지 6단계 파이프라인을 단계별로 따라가며 각 알고리즘이 "왜 그 자리에" 필요한지를 익힘.
   - LRU Cache 구현 패턴: Python OrderedDict 기반의 O(1) get/set, 그리고 캐시 키를 (id, comments, likes, watchCount, subMonths, lastWatchDays) 튜플로 잡아 입력이 바뀌면 자동 무효화되는 설계를 학습.
   - Heap Sort 재귀 heapify: sorted() 같은 내장 함수에 의존하지 않고 직접 max-heap을 만들어 O(n log n) 정렬을 구현하는 방식 학습.
   - Binary Search 활용: 점수 → 4그룹(🌱/👋/😊/🔥) 매핑처럼 경계값이 적은 경우에도 O(log k) 탐색을 적용하는 알고리즘 시연.
   - 외부 데이터 통합 전략: Google Takeout(comments.csv, watch-history.json)을 보조 입력으로 받아 YouTube Data API의 한계(채널당 100개 댓글 제한, 시청 기록 API 부재)를 극복하는 하이브리드 설계 패턴.
   - 코딩 테스트 알고리즘 패러다임 복습: 분할 정복, 탐욕법, 동적 계획법의 차이와 적용 케이스를 백준/LeetCode 문제로 점검.
