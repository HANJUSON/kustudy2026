# 주차별 학습목표
    트리의 재귀적 구조를 바탕으로 BST의 탐색·삽입·삭제 연산 원리와 평균/최악 시간 복잡도를 분석할 수 있다.

# 학습내용
    1. 트리의 재귀적 정의와 순회 복잡도

    트리는 루트 노드와 0개 이상의 서브트리로 구성된 재귀적 자료구조이다. 각 서브트리는 그 자체로 더 작은 트리이며, 이 반복적 부분 구조 덕분에 트리 알고리즘의 대부분을 재귀로 구현할 수 있다. 이는 분할 정복(divide and conquer)의 핵심 원리와 동일하다. 균형 이진 트리에서 모든 노드를 방문하는 순회의 점화식은 T(n) = 2T(n/2) + O(1)로 표현되며, 마스터 정리의 Case 1을 적용하면 T(n) = Θ(n)이 된다. 이는 각 노드를 정확히 한 번씩 방문하기 때문에 O(n)보다 더 나은 복잡도는 달성할 수 없다는 것을 의미한다.

    2. 이진 탐색 트리(BST)의 구조와 속성

    BST는 모든 노드 x에 대해 왼쪽 서브트리의 키는 x.key보다 작고, 오른쪽 서브트리의 키는 x.key보다 크다는 정렬 순서를 유지하는 이진 트리이다. 이 속성은 이진 탐색(binary search)과 동일한 원리로, 매 단계마다 탐색 공간의 절반을 제거할 수 있게 해준다.

    3. BST 탐색 연산

    TREE-SEARCH(x, k) 알고리즘은 현재 노드가 NIL이거나 키가 일치하면 반환하고, k < x.key이면 왼쪽 서브트리로, 그렇지 않으면 오른쪽 서브트리로 재귀 탐색한다. 각 레벨에서 정확히 하나의 비교 연산이 수행되며, 탐색 경로는 루트에서 리프까지 한 방향으로만 내려간다.

    4. BST 삽입 연산

    새 키 k를 삽입할 때는 먼저 탐색 경로를 따라 NIL에 도달할 때까지 내려간 후, 해당 위치에 새 노드를 리프로 연결한다. TREE-INSERT 알고리즘은 부모 포인터 y를 추적하면서 내려가다가, 마지막에 z의 부모를 y로 설정하고 z.key와 y.key를 비교하여 좌/우 자식으로 연결한다.

    5. BST 평균 시간 복잡도 분석

    데이터가 임의의 순서로 삽입되면 트리의 높이는 대략 log n이 된다. 탐색/삽입의 점화식은 T(n) = T(n/2) + O(1)이며, 마스터 정리를 적용하면 T(n) = Θ(log n)이 된다. 이는 정렬된 배열에서의 이진 탐색과 동일한 성능이다.

    6. BST 최악 시간 복잡도와 한계

    데이터가 정렬된 순서로 삽입되면(예: 1, 2, 3, 4, 5 순서) 트리는 연결 리스트처럼 한쪽으로 치우친 형태가 된다. 이 경우 점화식은 T(n) = T(n-1) + O(1)이 되어 T(n) = O(n)으로 퇴화한다. 즉 BST만으로는 최악의 경우 O(log n) 성능을 보장할 수 없다는 근본적인 한계가 존재한다.

    7. BST 삭제 연산 — 세 가지 케이스

    삭제는 탐색·삽입보다 복잡하며 세 가지 경우로 나뉜다. 첫째, 자식이 없는 노드(리프)는 단순히 제거한다. 둘째, 자식이 하나인 노드는 해당 노드를 자식으로 대체한다. 셋째, 자식이 둘인 노드는 중위 후계자(in-order successor), 즉 오른쪽 서브트리에서 가장 작은 노드를 찾아 삭제 대상 노드의 키를 대체한 후, 원래 중위 후계자를 삭제한다. 이 방식은 중위 후계자가 다음으로 큰 값이기 때문에 BST 속성을 보존한다. 모든 삭제 연산의 총 비용은 O(h)이며, 평균은 O(log n), 최악은 O(n)이다.

    8. 레드-블랙 트리(RBT)의 동기

    BST의 최악 O(n) 문제를 해결하기 위해 도입된 자료구조이다. 각 노드에 색상(빨강/검정) 1비트를 추가하고 5가지 규칙을 통해 트리가 항상 근사 균형을 유지하도록 보장한다. 이를 통해 높이가 h ≤ 2log₂(n+1) = O(log n)으로 제한된다.

    9. RBT의 5가지 속성

    (1) 모든 노드는 빨강 또는 검정이다. (2) 루트는 검정이다. (3) 모든 리프(NIL)는 검정이다. (4) 빨간 노드의 두 자식은 모두 검정이다(연속된 빨강 불허). (5) 임의의 노드에서 모든 하위 리프로 향하는 경로에 포함된 검정 노드의 수는 동일하다. 속성 4와 5가 협력하여 가장 긴 경로(빨강-검정 교대)가 가장 짧은 경로(전체 검정)의 최대 2배를 넘지 못하도록 제한한다.

    10. RBT가 O(log n)을 보장하는 수학적 근거

    흑색 높이 bh(x)를 x에서 리프까지 경로의 검정 노드 수로 정의하면, x를 루트로 하는 서브트리의 내부 노드 수는 최소 2^bh(x) - 1이다. 속성 4에 의해 경로상 최소 절반의 노드가 검정이므로 bh(root) ≥ h/2가 성립한다. 이 두 사실을 결합하면 n ≥ 2^(h/2) - 1이 되어 h ≤ 2log₂(n+1)임을 도출할 수 있다.

    11. RBT 삽입 시 이중 빨강(Double Red) 문제와 해결 전략

    새 노드는 항상 빨간색으로 삽입된다(속성 5 위반 방지). 부모도 빨간 경우 속성 4가 위반되며, 이를 이중 빨강(Double Red)이라 한다. 해결 전략은 삼촌(uncle) 노드의 색에 따라 구조 재편(Restructuring) 또는 재채색(Recoloring)으로 나뉜다.

    12. 구조 재편(Restructuring) — 삼촌이 검정일 때

    N, P, G 세 노드를 키 기준으로 정렬하고, 중앙값을 새 부모(검정)로, 나머지 둘을 자식(빨강)으로 배치한다. 이 연산은 로컬 연산으로 위쪽으로 전파되지 않으며 O(1)에 완료된다.

    13. 재채색(Recoloring) — 삼촌이 빨강일 때

    P와 U를 검정으로, G를 빨강으로 변경한다. G가 루트이면 강제로 검정으로 고정하고, G의 부모가 빨강이면 이중 빨강이 위쪽으로 전파될 수 있다. 단, 전파는 최대 O(log n)번 발생하므로 전체 삽입 비용은 O(log n)이다.

    14. RBT 전체 연산 복잡도

    탐색은 일반 BST와 동일하게 O(log n)이며, 삽입과 삭제는 각각 BST 연산 O(log n)에 고정(fix-up) 비용 O(log n)을 더해도 최종 O(log n)이다. fix-up은 최대 O(log n)번의 재채색과 최대 2번의 회전으로 이루어진다. 이로써 BST와 달리 최악의 경우에도 O(log n)을 보장한다.

    15. B-트리의 동기 — 디스크 I/O 병목

    데이터가 주기억장치를 초과해 디스크에 저장될 경우, HDD는 메모리 대비 약 50,000배 느리다. 10억 개 노드를 가진 레드-블랙 트리의 높이는 약 30으로, 30번의 디스크 접근은 HDD 기준 약 150ms가 소요된다. 목표는 디스크 접근 횟수, 즉 트리의 높이를 최소화하는 것이다.

    16. B-트리의 핵심 설계 철학

    각 노드가 하나의 디스크 블록(약 4KB)에 대응하도록 설계하여, 하나의 디스크 읽기에서 수백 개의 키를 한꺼번에 적재한다. 노드 내부 탐색은 CPU 연산으로 처리하므로 빠르다. 이 결과, 차수(order) 1000인 B-트리는 높이 3만으로 10억 개 이상의 키를 저장할 수 있다.

    17. B-트리 탐색

    루트에서 시작해 현재 노드의 키를 스캔하고, k가 두 키 사이에 있으면 해당 자식 포인터를 따라 내려간다. 키를 찾거나 리프에 도달할 때까지 반복하며, 레벨당 하나의 디스크 읽기가 발생한다.

    18. B-트리 삽입 — 분할 없는 경우와 분할 있는 경우

    대상 리프에 공간이 있으면 정렬 순서에 맞춰 단순 삽입한다. 리프가 가득 찬 경우, 임시 오버플로우 노드에서 중앙값을 부모로 올리고(promote) 나머지를 좌우 자식으로 분할한다. 부모도 가득 찬 경우 분할이 위쪽으로 연쇄 전파되며, 극단적인 경우 루트가 분할되어 트리가 한 레벨 높아진다.

    19. B-트리 삭제의 복잡성

    삭제는 B-트리에서 가장 복잡한 연산이다. 리프 노드의 경우 여유 키가 있으면 단순 삭제, 형제에서 차용(rotate through parent) 가능하면 부모 회전, 형제도 최솟값이면 병합(merge) 후 부모 키를 아래로 끌어내린다. 내부 노드의 경우 중위 전임자 또는 후계자로 대체한 뒤 리프 삭제 케이스로 환원한다.

    20. BST, RBT, B-트리 비교

    BST는 평균 O(log n)이지만 최악 O(n)이며 균형 보장이 없다. RBT는 색상 1비트를 추가하여 모든 연산을 최악의 경우에도 O(log n)으로 보장하는 범용 인메모리 균형 트리이다. B-트리는 노드당 수백~수천 개의 키를 저장하여 높이를 극단적으로 낮추고, 디스크 접근 횟수를 최소화하는 데 특화된 구조로 DBMS와 파일 시스템의 핵심 자료구조로 활용된다.
# 상세활동 내용
    BST에서 RBT까지 — 삽입 시퀀스 비교 실습

    빈 BST와 빈 RBT에 동일한 키 시퀀스 {7, 3, 8, 1, 5}를 순서대로 삽입하며 두 자료구조의 차이를 직접 체험하는 활동이다.

    먼저 BST에 7, 3, 8, 1, 5를 순서대로 삽입하고 최종 트리를 종이에 그린다. 각 삽입 단계마다 비교 횟수와 삽입 위치를 기록하고, 완성된 트리의 높이를 측정한다. 이어서 동일한 시퀀스를 오름차순 정렬 순서인 {1, 2, 3, 4, 5}로 바꿔 삽입하고, 트리가 연결 리스트처럼 퇴화하는 현상을 직접 확인한다. 두 경우의 높이를 비교하여 BST가 삽입 순서에 얼마나 민감한지를 체감한다.

    다음으로 RBT에 {7, 3, 8, 1, 5}를 순서대로 삽입한다. 각 삽입 단계에서 새 노드를 빨간색으로 추가한 뒤, 이중 빨강이 발생하는지 확인한다. 4번째 노드(1) 삽입 시 N=1, P=3, U=8이며 삼촌이 빨간색이므로 재채색을 적용하고, 5번째 노드(5) 삽입 시 N=5, P=3, U=8이며 삼촌이 이제 검정색이므로 구조 재편을 적용하는 과정을 그림으로 그린다. 각 단계가 끝날 때마다 5가지 속성이 모두 만족되는지 체크리스트로 검증한다.

    마지막으로 최종 BST와 최종 RBT의 높이를 비교하고, RBT의 높이가 항상 2log₂(n+1) 이하임을 n=5일 때 수치로 계산하여 확인한다.
