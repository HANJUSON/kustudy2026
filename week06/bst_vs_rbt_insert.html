빈 BST에 {7, 3, 8, 1, 5}를 순서대로 삽입한 최종 트리. 임의 순서이므로 균형에 가깝게 형성된다.

② BST 삽입 정렬 입력

동일 BST에 {1, 2, 3, 4, 5}를 오름차순으로 삽입 → 한쪽으로 치우친 연결 리스트로 퇴화, 높이 O(n).

4

O(n)

③ RBT 삽입 색상 1비트

동일 시퀀스 {7, 3, 8, 1, 5}를 RBT에 삽입. 재채색·구조 재편으로 균형이 강제 유지된다.

빨강 노드 검정 노드
2
트리 높이 (h)
O(log n)
최악 보장

RBT 5속성 체크리스트 — 삽입 완료 후 전수 검증

속성 4와 5가 협력하여 가장 긴 경로(빨강-검정 교대)가 가장 짧은 경로(전체 검정)의 최대 2배를 넘지 못하도록 제한한다.

  • ✓속성 1. 모든 노드는 빨강 또는 검정이다. → 5개 노드 모두 색 지정 완료.
  • ✓속성 2. 루트는 검정이다. → 루트 7은 검정.
  • ✓속성 3. 모든 리프(NIL)는 검정이다.
  • ✓속성 4. 빨간 노드의 두 자식은 모두 검정이다(연속된 빨강 불허). → 빨강 1·5의 자식은 NIL(검정).
  • ✓속성 5. 임의의 노드에서 모든 하위 리프로 향하는 경로의 검정 노드 수가 동일하다. → 흑색 높이 bh(root)=2 일정.

높이 한계 검증 n = 5

RBT 높이는 항상 h ≤ 2log₂(n+1). n=5일 때 수치로 확인.

h ≤ 2·log₂(n+1)
= 2 · log₂(5+1)
= 2 · log₂(6)
= 2 · 2.585
= 5.17 이하
실제 RBT 높이 = 2 ≤ 5.17 ✓
근거: 흑색 높이 bh(root) ≥ h/2 이고 n ≥ 2^(h/2) − 1 이므로 h ≤ 2log₂(n+1) 이 성립한다.

세 트리 높이 비교 — 삽입 순서·균형 여부에 따른 높이 차이

동일한 노드 수(n=5)라도 균형 보장 여부에 따라 높이가 2배 이상 벌어진다. 막대 길이가 곧 최악 탐색 비용이다.
