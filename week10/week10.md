# 주차별 학습목표
    운영체제의 대용량 저장장치 구조와 I/O 스케줄링 알고리즘, RAID 등 스토리지 관리 전반을 이해하고 실제 시스템 설계에 적용할 수 있다.
# 학습내용
    1. HDD의 물리적 구조(플래터, 트랙, 섹터, 실린더)와 RPM별 성능 특성을 학습했다.

    2. 접근 시간(Access Time)이 탐색 시간, 회전 지연, 전송 시간으로 구성됨을 이해했다.

    3. NVM(SSD)의 NAND Flash 특성인 페이지 단위 읽기/쓰기, 블록 단위 삭제 구조를 파악했다.

    4. FTL, 가비지 컬렉션, 웨어 레벨링, TRIM 등 SSD 내부 관리 알고리즘을 학습했다.

    5. FCFS, SCAN, C-SCAN, LOOK, C-LOOK 등 HDD 스케줄링 알고리즘의 동작 원리와 성능을 비교했다.

    6. LBA(논리 블록 주소) 기반의 주소 추상화 개념과 저장장치 연결 인터페이스(SATA, NVMe, SAS 등)를 이해했다.

    7. 패리티, ECC, CRC 등 오류 감지 및 수정 기법의 원리와 HDD/SSD 적용 방식을 학습했다.

    8. 디스크 포매팅(저수준 → 파티셔닝 → 논리 포매팅), 부트 블록, 불량 섹터 관리 방법을 학습했다.

    9. 스왑 공간의 목적, 구현 방식, 리눅스 내부 구조(스왑 맵)를 이해했다.

    10. RAID 레벨(0·1·4·5·6·10)의 구성 방식, 용량 효율, 신뢰성, 성능 특성을 비교 분석했다.
# 상세활동 내용

    1. HDD 구조 다이어그램을 보며 플래터, 헤드, 디스크 암의 물리적 관계를 그림으로 정리했다.

    2. 7,200 RPM 디스크 기준으로 평균 접근 시간(약 12.18ms)을 직접 계산하며 탐색 시간의 비중을 확인했다.

    3. NAND Flash의 덮어쓰기 불가 특성과 가비지 컬렉션 동작 흐름을 블록 상태 변화 예시로 이해했다.

    4. FCFS와 SCAN 알고리즘을 동일한 요청 큐(98, 183, 37, …)에 적용해 이동 거리(640 vs 236)를 비교했다.

    5. C-SCAN과 LOOK/C-LOOK의 차이점을 탐색 방향성 및 끝점 도달 여부 기준으로 정리했다.

    6. RAID 4의 패리티 디스크 병목 문제를 분석하고, RAID 5의 분산 패리티가 이를 해결하는 원리를 학습했다.

    7. RAID 0+1과 RAID 10의 단일 디스크 장애 시 복구 범위 차이를 비교하며 RAID 10의 우수성을 확인했다.

    8. 리눅스의 Deadline, NOOP, CFQ 스케줄러의 특성과 적합한 사용 환경을 표로 비교 정리했다.

    9. NAS와 SAN의 접근 레벨(파일 vs 블록), 프로토콜, 비용, 성능 차이를 구분하여 학습했다.

    10. 랩 실습 코드에서 FCFS, SCAN, C-SCAN, LOOK, C-LOOK 알고리즘을 Python으로 구현하며 결과값을 검증했다.
